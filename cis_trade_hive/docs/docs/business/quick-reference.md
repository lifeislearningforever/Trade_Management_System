# CisTrade Quick Reference Card

## Portfolio Workflow

```
Create → Draft → Submit → Pending → Approve → Active → Close → Inactive
                              ↓
                          Reject → Edit → Resubmit
```

## Key Actions by Status

| Status | Available Actions |
|--------|-------------------|
| **Draft** | Edit, Submit for Approval |
| **Pending Approval** | Approve, Reject (Checker only) |
| **Active** | Close Portfolio |
| **Inactive** | Reactivate Portfolio |
| **Rejected** | Edit, Resubmit |

## User Roles

**Maker**: Creates & submits portfolios
**Checker**: Approves/rejects (different user - Four-Eyes)

## Quick Tips

✅ Required fields: Code, Name, Currency, Manager
✅ Use search to find portfolios quickly
✅ Provide reasons when closing/rejecting
✅ Cannot edit Active portfolios
✅ Cannot approve your own work

## Search Shortcuts

- Search by: Code, Name, Manager
- Filter by: Status, Currency
- Export: Download CSV button

## Common Tasks

**Create Portfolio**: Portfolios → Create New
**Submit for Approval**: Detail page → Submit button
**Approve**: Pending Approvals → Approve
**Close**: Active portfolio → Close button

## Need Help?
